Everything included

Stop wiring together an SSH client, a file manager, a remote editor, and an AI helper. OxideTerm covers all of them — and they share state so you never copy-paste a path between windows.

🖥️ Local & remote terminal

Local PTY (zsh, bash, fish, pwsh, Git Bash, WSL2) and SSH in the same window. Split panes horizontally or vertically. Broadcast keystrokes to multiple panes. Adaptive 3-tier WebGL renderer (Boost 120 Hz / Normal 60 Hz / Idle <15 Hz). 100 000-line scroll buffer.

🔁 Smart reconnect

When the SSH link drops, a 30-second grace period probes the old connection before doing anything destructive. If it recovers, TUI apps (vim, htop, yazi…) keep running. On full reconnect, forwards, transfers, and IDE state are all restored automatically.

📁 SFTP file manager

Dual-pane browser: local on the left, remote on the right. Drag-and-drop, batch rename, real-time transfer queue with speed and ETA. Rich previews: images, video, audio, PDF, hex, 30+ language code highlighting.

📂 Local file manager

Standalone local pane with bookmarks and the same rich preview engine. Also integrates as the left-side pane of SFTP. macOS-aware volume picker; no extra app needed for quick file ops.

📝 Remote IDE

CodeMirror 6 editor over SFTP — no server installation needed by default. Lazy-loaded file tree with Git-status badges, project-wide full-text search, multi-tab with LRU eviction, pin tabs, and optimistic-lock conflict resolution. An optional 1 MB Rust agent on Linux unlocks extra power.

🔀 Port forwarding

Local (-L), remote (-R), and dynamic SOCKS5 (-D) forwards — all managed through the UI, restored on reconnect, and reported when they die unexpectedly. Lock-free I/O: channel owned by a single task, no mutex on the hot path.

🗺️ Topology graph

D3-force visualisation auto-generated from saved connections and ProxyJump chains. Dijkstra path calculation, status colour-coding, double-click context menu. Unlimited ProxyJump hops with cascading failure propagation.

🤖 OxideSens AI

Inline quick panel (⌘I) and persistent sidebar chat. 40+ built-in tools, MCP server integration, RAG knowledge base, and an autonomous agent (plan → execute → review) — compatible with OpenAI, Ollama, DeepSeek, and any /v1/chat/completions endpoint.

🧩 Plugin system

Runtime ESM bundles loaded from ~/.oxideterm/plugins/. 8 API namespaces, 24 UI Kit components, frozen context objects, error boundaries, auto-disable after repeated failures. Shared React/Zustand/lucide via window.__OXIDE__.

🎨 Themes & appearance

30+ built-in themes (Dracula, Nord, Catppuccin, Tokyo Night…). Visual editor covering all 22 xterm.js colour fields and 19 app-chrome CSS variables. Auto-derive UI colours from the terminal palette. Live preview. Background image gallery with per-tab opacity/blur.

📊 Resource profiler

Reads /proc/stat, meminfo, loadavg, net/dev over a persistent SSH shell channel — single channel, avoids MaxSessions exhaustion. Degrades gracefully to RTT-only on non-Linux or failures.

🎬 Session recording

Records to asciicast v2 NDJSON. Pause/resume, idle-gap limiting, optional input capture. Built-in CastPlayer modal for playback — no external tools needed.

Under the hood

A few architectural choices that make a real difference at runtime — not marketing copy, these are in the source.

⚡

Pure Rust SSH — no OpenSSL

The entire SSH stack uses russh 0.54 compiled against the ring crypto backend. No C dependencies in the SSH path: smaller attack surface, easier cross-compilation, no OpenSSL version drift. Supports ChaCha20-Poly1305, AES-GCM, Ed25519, RSA, ECDSA, OpenSSH certificates, keyboard-interactive 2FA, and a custom AgentSigner for system SSH agents on macOS, Linux, and Windows.

🔁

Reconnect orchestrator with grace period

An 8-stage pipeline manages every reconnect: queued → snapshot → grace-period → ssh-connect → await-terminal → restore-forwards → resume-transfers → restore-ide → done. The grace period (30 s, probing every 3 s) detects whether the old SSH link is still alive. If it recovers, destructive reconnect is skipped and TUI applications — vim, htop, yazi — are preserved. All reconnect logic goes through the orchestrator; nothing reconnects directly from event handlers.

🔗

SSH connection pool & multiplexing

A reference-counted SshConnectionRegistry shares one physical SSH connection across every terminal pane, SFTP session, and port forward opened on the same host. Independent state machines per connection (connecting → active → idle → link_down → reconnecting). Idle timeout 30 min, keep-alive 15 s, WsBridge heartbeat 5 min (tolerates macOS App Nap).

🧭

Node-first architecture (NodeRouter)

The frontend never passes a sessionId or connectionId to the backend. Everything is addressed by a stable nodeId; the NodeRouter resolves it to the right ConnectionEntry atomically. When SSH reconnects and a new connectionId is issued, SFTP and IDE operations are unaffected — the nodeId stays constant.

📡

Dual-plane binary protocol

Terminal I/O travels over a WebSocket using a custom binary wire protocol: [Type:1B][Length:4B BE][Payload:nB]. Four frame types: Data, Resize, Heartbeat, Error. Maximum payload 16 MB. Zero JSON serialisation overhead on the hot path. Control commands (SFTP ops, connection management, settings) use Tauri IPC on a separate plane.

🗂️

16-store Zustand state architecture

State is split across 16 specialised stores — sessionTreeStore (user intent), appStore (SSH fact layer), ideStore, localTerminalStore, reconnectOrchestratorStore, transferStore, pluginStore, aiChatStore, agentStore, and more. Strict dual-store sync invariants prevent stale reads. Consumers always read from appStore.connections, never from user-intent copies.

OxideSens — AI that works in your terminal

Two modes, one goal: help you get things done in your existing SSH & local environment without switching apps.

💬 Sidebar & inline chat

Persistent sidebar conversation with context from the terminal buffer, IDE files, SFTP paths, and Git status. Quick inline panel (⌘I) injects responses via bracketed paste. Compatible with OpenAI, Ollama, DeepSeek, OneAPI — bring your own endpoint.

🦾 Autonomous agent

Three-phase loop: Plan (LLM builds a step-by-step plan), Execute (tool calls with per-tool approval gating), Review (optional reviewer model checks progress). Autonomy levels: Supervised / Balanced / Autonomous. Dangerous commands always require approval. Resume from any prior round.

🔧 40+ built-in tools

terminal_exec, local_exec, batch_exec, read_file, write_file, sftp_ls/get/put, open_tab, open_session_tab, search_docs, and more. Tools are gated to connectionState === 'active' — the backend enforces state gating as a second layer.

🔌 MCP (Model Context Protocol)

Connect external MCP servers over stdio or SSE. Tools are auto-namespaced as mcp::server::tool and merged into the agent's tool list. Auto-retry with exponential backoff. Managed in Settings — no code changes needed.

📚 RAG knowledge base

Import Markdown/TXT runbooks and SOPs into scoped collections (global or per-connection). Hybrid retrieval: BM25 keyword index + vector cosine similarity fused with RRF. Markdown-aware chunking by heading hierarchy. CJK bigram tokeniser. AI calls search_docs automatically during conversations — no manual paste.

🔑 Privacy-first key handling

API keys stored in the OS keychain (com.oxideterm.ai service). On macOS, reading a key requires Touch ID (via LAContext) — no data-protection entitlements needed. Keys are cached in memory after first auth; biometric is only prompted once per session.

Why OxideTerm?

Common pain points with existing workflows — and how OxideTerm addresses them.

My SSH client can't open a local shell
Hybrid engine — local PTY and SSH sessions coexist in one window, one session tree.
Dropping connection means losing everything
Grace-period reconnect — probes the old link for 30 s before doing anything destructive; restores forwards, transfers, and IDE state automatically.
Remote editing requires installing a server binary
IDE mode over SFTP — works on any SSH host by default. No server-side installation needed; optional lightweight agent for extra features on Linux.
Every tool opens a separate connection
SSH multiplexing — terminal, SFTP, port forwards, and the profiler all share one physical SSH connection via a reference-counted pool.
Credentials end up in config files or a cloud account
Local-first security — passwords and API keys stored in the OS keychain only; config files hold reference IDs, never secrets. No mandatory account or cloud sync.
AI knows nothing about my servers
Context-aware AI — OxideSens reads terminal buffer, open IDE files, SFTP paths, and Git status. RAG knowledge base lets you inject runbooks and SOPs. MCP extends it with any external tool server.

Reconnect pipeline in detail

Most terminal clients reconnect bluntly — kill the old session, start a new one. OxideTerm runs a structured 8-stage pipeline with a grace period designed specifically to preserve long-running TUI apps.

Snapshot captures: active port forwards, in-flight SFTP transfers, IDE project path, open tabs, and dirty file contents. If grace period succeeds — SSH keepalive responds within the window — stages 4–8 are skipped entirely and your TUI app never sees a SIGHUP.

Security architecture

Credentials never touch config files. Everything sensitive goes through the OS keychain or is encrypted before leaving memory.

🔐 OS keychain — always

SSH passwords stored under com.oxideterm.ssh; AI API keys under com.oxideterm.ai. Config files contain only keychain reference IDs — never the raw secrets. Uses the keyring crate (macOS Keychain, Windows Credential Manager, Linux libsecret).

👆 Touch ID for AI keys (macOS)

Key reads are gated behind LAContext biometric authentication. No data-protection entitlements required. Keys cached in memory after first auth — biometric is only prompted once per session, not on every message.

🗝️ .oxide encrypted export

Portable backup format: ChaCha20-Poly1305 AEAD + Argon2id KDF (256 MB memory, 4 iterations). Optional private key embedding. SHA-256 integrity check. Pre-flight analysis shows auth types and flags missing keys before export.

🧹 Memory hygiene

Sensitive buffers (passwords, key material) cleared via zeroize — Rust drops do not guarantee zero-fill. Rust's ownership model prevents use-after-free and data races at the language level.

🔏 Host key TOFU

First-connect fingerprint pinned in ~/.ssh/known_hosts. Host key changes are rejected with an explicit UI warning — no silent MITM acceptance.

🧱 Plugin sandbox

All plugin context objects are frozen with Object.freeze(). Proxy ACL blocks un-permitted APIs. Circuit breaker auto-disables plugins after repeated errors. IPC whitelist restricts which Tauri commands plugins can call.

Authentication methods

Every standard SSH auth method, plus certificates and hardware agents.

Password
Stored in OS keychain; never written to disk in plaintext.
SSH Key
RSA, Ed25519, ECDSA — auto-scans ~/.ssh/id_*.
SSH Agent
Custom AgentSigner via Unix socket or Windows named pipe. Replayed automatically on reconnect.
Certificate
OpenSSH certificates with CA verification.
Keyboard-Interactive
2FA / MFA challenge-response; handles multi-step prompts.
Known Hosts
TOFU + ~/.ssh/known_hosts; rejects changed fingerprints.

Quick start (build from source)

Pre-built binaries are on the Releases page. To build locally you need Rust 1.75+, Node 18+, and platform tooling. 

git clone https://github.com/AnalyseDeCircuit/OxideTerm.git
cd OxideTerm
pnpm install

# Full app (Rust backend + React frontend)
pnpm tauri dev

# Production bundle
pnpm tauri build

# Lightweight — strip local PTY (e.g. for CI / mobile)
cd src-tauri && cargo build --no-default-features --release

Platform prerequisites: macOS needs Xcode CLT; Windows needs VS C++ Build Tools; Linux needs build-essential libwebkit2gtk-4.1-dev libssl-dev.
